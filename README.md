# Аудит рабочих задач — лендинг

Интерактивная страница по Excel-файлу **Аудит_рабочих_задач** (данные в `src/data/audit.json`).

## Живая версия для просмотра

После деплоя на **Vercel** сюда вставьте вашу короткую ссылку, например: `https://audit-landing-xxx.vercel.app` и отправьте её ПДК-куратору (открыть в браузере, без GitHub).

## Быстрый вход (Windows)

| Файл | Когда использовать |
|------|---------------------|
| `ОТКРЫТЬ_ЛЕНДИНГ.bat` | Убить порт 3000 → `npm run build` → браузер → `npm run start`. |
| `ОБНОВИТЬ_ДАННЫЕ.bat` | Обновить `audit.json` через `..\scripts\export_audit_v2.py` (родительская папка «Аналитик данных»). |
| `СОБРАТЬ_ФИНАЛ.bat` | Редкая финальная сборка; перед `start` также чистится порт 3000. |

Подробнее: **`docs/WORKFLOW.md`**.

## Запуск вручную

```bash
cd audit-landing
npm install
npm run dev
```

[http://localhost:3000](http://localhost:3000).

> **Windows + длинный путь OneDrive:** при ошибке Turbopack используйте `npm run build` → `npm run start` или короткий путь `subst` к папке проекта.

## Обновление данных

1. Положите `.xlsx` в корень родительской папки (**Аналитик данных**) или поправьте путь в скрипте.
2. Выполните `python scripts/export_audit_v2.py` из родительской папки или `ОБНОВИТЬ_ДАННЫЕ.bat`.
